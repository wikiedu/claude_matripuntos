## 2. Navegación

### Bottom navigation bar

```
🏠 Inicio  |  ✅ Tareas  |  [➕]  |  📅 Calendario  |  🏆 Logros
```

**Especificaciones:**
- Posición: `fixed` al fondo en móvil, siempre visible
- Fondo: `bg-card` con borde superior sutil
- Tab activo: label en `accent-amber`, icono ligeramente realzado
- Tabs inactivos: icono + label en `text-muted`

**Botón ➕ central:**
- Círculo de 44×44px con `background: linear-gradient(135deg, #f59e0b, #d97706)`
- `margin-top: -20px` para elevarlo sobre la barra
- `box-shadow: 0 4px 12px rgba(245,158,11,0.4)`
- Al pulsarlo: abre un bottom sheet con dos opciones → "Nueva actividad" / "Registrar tarea"
- El sheet reemplaza los flujos actuales de RequestActivity y el log de tareas

**Módulos secundarios** (Analytics, Settings, Historial):
- Analytics: accesible desde un icono de gráfica en el header del Dashboard
- Settings: icono ⚙️ en el header
- Historial: desde dentro de la sección de Puntos o Settings

### Migración desde la navegación actual

La app actual usa una mezcla de `useNavigate` y `currentView` state en Dashboard. En v1.1:
- Cada tab del bottom nav navega a su ruta de React Router
- Se elimina el `currentView` state del Dashboard
- `RequestActivity` y `RequestInbox` pasan a ser rutas propias accesibles desde el ➕ y desde Tareas respectivamente

## 4. Mood del día

### Comportamiento

- El mood aparece como badge emoji (14×14px) superpuesto en la esquina inferior derecha del avatar
- Tap en el avatar → bottom sheet con selector de 8 estados
- Sin puntos, sin notificaciones al cambiar mood
- El mood del partner aparece como línea contextual bajo el saludo en el Dashboard: *"[Nombre] está [emoji] [label] hoy"* — solo si el partner tiene mood definido ese día

### 8 estados

| Emoji | Label |
|---|---|
| 😊 | Animado/a |
| 😴 | Cansado/a |
| 😤 | Estresado/a |
| 🥰 | Contento/a |
| 😰 | Agobiado/a |
| 🧘 | Tranquilo/a |
| 😂 | De buen humor |
| 🤯 | Saturado/a |

### API

```
PUT /api/profile/me  { currentMood: "😊" }
```

El mood del partner se obtiene añadiendo `partnerMood` a la respuesta del endpoint `GET /api/profile/me`, que ya devuelve datos del couple. Alternativa: nuevo campo en `GET /api/points/balance` que ya carga info de ambos usuarios. Se elige la primera opción — menos endpoints nuevos.

## 5. Frase motivacional diaria

### Comportamiento

- Una frase por día, determinada por `dayOfYear % FRASES.length`
- Aparece en el Dashboard como card amber sutil debajo del balance
- Sin API — array de constantes en el frontend

### Implementación

```typescript
// src/frontend/src/utils/dailyPhrase.ts
const FRASES = [
  "El equilibrio no es hacer lo mismo, es valorar lo que hace el otro.",
  "Una pareja que negocia junta, crece junta.",
  // ... ~60 frases total
]

export function getDailyPhrase(): string {
  const now = new Date()
  const start = new Date(now.getFullYear(), 0, 0)
  const diff = now.getTime() - start.getTime()
  const dayOfYear = Math.floor(diff / (1000 * 60 * 60 * 24))
  return FRASES[dayOfYear % FRASES.length]
}
```

### Frases (60 a definir en implementación)

Tono: mezcla de humor, motivación y contexto de pareja/hogar. Ejemplos:
- *"El equilibrio no es hacer lo mismo, es valorar lo que hace el otro."*
- *"Una pareja que negocia junta, crece junta."*
- *"Hoy es buen día para sorprender al otro con algo inesperado."*
- *"Los puntos no mienten. El historial tampoco."*
- *"Pequeñas tareas, grandes gestos."*

## 6. Onboarding mejorado

### Cambio

Se añade un **paso 0** (pantalla pre-registro) que explica la propuesta de valor antes de pedir datos:
- 3 bullets visuales: qué es Matripuntos, cómo funciona, por qué es justo
- CTA: "Crear cuenta" + "¿Ya tienes código de pareja? Únete"
- Sin modo demo — demasiado scope para v1.1

### Flujo actualizado

```
Paso 0 (PVP) → Paso 1 (datos usuario) → Paso 2 (perfil) → Paso 3 (familia) → Paso 4 (pareja/invitar)
```

El paso 4 permite continuar sin invitar ("Lo haré después") — el usuario queda con `couple` incompleto y puede invitar desde Settings.

## 9. Consideraciones técnicas

- **Prisma migration:** Añadir `avatarEmoji`, `avatarColor`, `theme`, `currentMood`, `moodUpdatedAt` a `UserProfile`. Non-breaking (todos con defaults).
- **Tailwind config:** Añadir tokens de color custom para no duplicar valores hex.
- **CSS variables:** Para el dark/light toggle se usa una clase en `<html>` (`class="dark"` o `class="light"`) y variables CSS que cambian según la clase. Tailwind ya soporta esto con `darkMode: 'class'`.
- **Bottom nav routing:** Refactor del Dashboard para eliminar `currentView` state y usar React Router directamente.
- **No breaking changes en API:** Todo lo nuevo son campos opcionales en UserProfile + un PUT endpoint ya existente.
