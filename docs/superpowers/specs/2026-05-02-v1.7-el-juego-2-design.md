# v1.7 · El Juego (segundo round) — Spec

**Fecha:** 2026-05-02
**Versión:** v1.7
**Branch:** `feature/v1.7-el-juego-2`
**Tag al merge:** `v1.7`

## 1. Goal

Convertir Matripuntos en una experiencia con **motivación intrínseca y compromiso a largo plazo**. v1.6 introdujo personalidad (mood, frase, avatares); v1.6.x cerró privacidad y auditoría. v1.7 cierra el círculo añadiendo capa de juego que premia la **constancia, equilibrio y colaboración** sin convertir el equilibrio doméstico en una compra de puntos.

## 2. Problemas a resolver

1. **No hay reward loop visible.** El user gana puntos, ve un saldo neto, pero no hay "siguiente meta" inmediata después de aceptar una actividad.
2. **Achievements son vaporware.** Hay tabla en BD, panel en UI, pero no se desbloquean condiciones reales en ningún flujo.
3. **Constancia no se reconoce.** Una pareja activa 8 semanas seguidas tiene la misma señal visible que una que entró ayer.
4. **No hay retos en pareja.** El producto es transaccional individual ("yo gano X, tú ganas Y") cuando el valor real es el equilibrio.
5. **Engagement decae a las 3-4 semanas.** Sin gancho de re-engagement, los users dejan de loggear tareas.

## 3. Pillars de diseño

- **Cooperación > competición**: los logros más visibles son de pareja, no individuales. La pareja gana junta.
- **Reconoce intención, no volumen**: registrar 3 tareas equilibradas vale más que 30 disparejas.
- **Streaks blandos**: "32 semanas activos" se preserva con 1 acción/semana, no requiere 7/7. Ruptura con margen 24h.
- **Niveles aspiracionales pero accesibles**: progresión cada 2-4 semanas para users medios, no horas de grind.
- **Sin pay-to-progress**: premium nunca da puntos, solo personalización (badges custom, themes, frases extra).

## 4. Scope v1.7

### 4.1 Sistema de niveles de pareja
- Modelo `CoupleLevel` con XP acumulado de pareja (suma de todas las transactions positivas).
- 10 niveles: Vecinos · Amigos · Cómplices · Equipo · Aliados · Tribu · Familia · Hogar · Refugio · Vida.
- Cada nivel desbloquea: tema visual sutil del Dashboard (gradiente sutil), avatar marco, frases especiales.
- XP requerido escalado: 100 · 300 · 700 · 1500 · 3000 · 6000 · 12000 · 24000 · 50000 · 100000.
- Visualización: barra discreta en AppHeader + animación de level-up al cruzar threshold.

### 4.2 Sistema de logros real (achievementEngine activo)
**6 categorías × ~5 logros cada una = ~30 logros total**:
- **Primer paso**: primera actividad aceptada, primera tarea verificada, primer mood, primera frase vista, primera negociación.
- **Constancia**: 7 días consecutivos · 30 días · 90 días · 365 días · 1000 días.
- **Equilibrio**: |saldo| ≤ 5 durante 7d · 30d · 90d · 180d · 365d.
- **Colaboración**: 10 tareas verificadas mutuamente · 50 · 100 · 250 · 500.
- **Diversidad**: 5 categorías distintas en una semana · todas las categorías en un mes · 100 actividades de tipos distintos.
- **Hitos**: aniversario 1 año en la app · 100 actividades aceptadas · 500 puntos a favor en una semana.

Cada logro tiene `condition` JSON evaluada al final de cada acción relevante (post-PointsTransaction commit). Engine reactivo con backpressure (si 10 logros se desbloquean a la vez, queue async).

### 4.3 Streaks de pareja
- `daily_streak`: días consecutivos con ≥1 acción de cualquier user de la pareja. Margen 24h+30min de gracia.
- `weekly_streak`: semanas consecutivas con ≥3 acciones distribuidas en al menos 3 días.
- Visible en Dashboard: "🔥 12 días" con animación discreta.
- Reset visible (no silente) con cita motivacional al perder streak: "Volver a empezar es de valientes 💪".

### 4.4 Retos semanales de pareja
- Lunes 00:00 (TZ usuario) se genera un challenge nuevo automático para la pareja según su nivel.
- Tipos: "Equilibrar saldo a ≤5 esta semana" · "Verificar 10 tareas mutuamente" · "Probar 3 categorías nuevas" · "Sin disputed logs" · "Una actividad de 'alto impacto' aceptada por ambos".
- Reward: badge temporal de la semana + 50-200 XP de pareja (según dificultad).
- Indicador de progreso visible en Dashboard top.
- Si la pareja completa, animación celebratoria. Si no completa, no penaliza — solo desaparece.

### 4.5 Replay del histórico (re-engagement)
- "Hace 1 año…" / "Vuestro mejor día" / "Récord de equilibrio" — cards aparecen ocasionalmente en Dashboard.
- Computado lazy: query semanal pre-cacheada en `CoupleScore`.
- Click → CoupleStory mini-página con stats agregadas + foto avatar pareja.

### 4.6 Notificaciones de re-engagement (opt-in)
- Push: "Hace 4 días que no ves a tu Matripuntos. Tu pareja registró 3 tareas. Ve a apoyarla 💕"
- Web push API + service worker (NO requerimos PWA todavía, solo web push).
- Frequency cap: 1 push máximo cada 48h. Settings → toggle.

## 5. Data model (additions)

```prisma
model CoupleLevel {
  id          String   @id @default(cuid())
  coupleId    String   @unique
  couple      Couple   @relation(fields: [coupleId], references: [id], onDelete: Cascade)
  xp          Int      @default(0)
  level       Int      @default(1)
  updatedAt   DateTime @updatedAt
}

model CoupleStreak {
  id              String   @id @default(cuid())
  coupleId        String   @unique
  couple          Couple   @relation(fields: [coupleId], references: [id], onDelete: Cascade)
  dailyStreak     Int      @default(0)
  weeklyStreak    Int      @default(0)
  lastActivityAt  DateTime?
  longestDaily    Int      @default(0)
  longestWeekly   Int      @default(0)
}

model CoupleChallenge {
  id          String   @id @default(cuid())
  coupleId    String
  couple      Couple   @relation(fields: [coupleId], references: [id], onDelete: Cascade)
  weekStart   DateTime
  type        String   // 'balance' | 'verify' | 'diversity' | 'no_dispute' | 'high_impact'
  config      String   // JSON con goal específico
  status      String   @default("active") // active | completed | expired
  progress    Int      @default(0)
  goal        Int
  reward_xp   Int
  completedAt DateTime?
  createdAt   DateTime @default(now())

  @@unique([coupleId, weekStart])
  @@index([coupleId, status])
}

model AchievementDefinition {
  id          String   @id  // 'first-activity' | 'streak-7' | etc.
  category    String
  name        String
  description String
  rarity      String   // common | rare | epic | legendary
  icon        String   // emoji
  conditionType String  // 'count' | 'streak' | 'threshold' | 'diversity'
  conditionConfig String  // JSON
  rewardXp    Int
  isHidden    Boolean  @default(false)
}

// CoupleAchievement y UserAchievement ya existen en v1.x — se reaprovechan.
```

## 6. API routes nuevas

```

GET  /api/gamification/streak            → { daily, weekly, longestDaily, longestWeekly }
GET  /api/gamification/challenge         → challenge actual de la semana + progreso
POST /api/gamification/challenge/skip    → premium-only, regenera challenge
GET  /api/gamification/replay            → cards "hace 1 año" / "mejor día" disponibles ahora
POST /api/gamification/replay/:id/seen   → marca card como vista (no repetir 30d)
PUT  /api/notifications/settings         → toggle web push + retention frequency
POST /api/notifications/subscribe        → web push subscription
```

Achievements existentes (`/api/achievements`, `/api/achievements/user`) se mantienen.

## 10. Decisiones brainstorm (resumen)

| # | Tema | Decisión | Por qué |
|---|---|---|---|
| 1 | Niveles | Por pareja (no individuales) | Pillar "cooperación > competición" |
| 2 | XP fórmula | XP = sum(PointsTransaction.amount > 0) | Reaprovecha sistema existente |
| 3 | Streaks | Daily + weekly con margen | Diario único es frágil; weekly demasiado laxo |
| 4 | Challenge generation | Server-side cron lunes 00:00 user TZ | Predictible y testable |
| 5 | Achievement engine | Reactivo post-commit, async | Latencia transparente |
| 6 | Notifications channel | Web Push primero, no PWA full | Escala incremental |
| 7 | Premium hooks | Custom badges + skip challenge | No P2W |
| 8 | Replay frequency | Lazy compute semanal pre-cacheado | Fast loads |
| 9 | Hidden achievements | Solo legendary, max 3 | Sorpresa sin frustración |
| 10 | Reset streak UX | Cita motivacional, no negative | Pillar "intención > volumen" |

## 12. Riesgos y mitigaciones

| Riesgo | Mitigación |
|---|---|
| Engine de achievements lento bajo concurrencia | Queue + caché de definitions + index en `CoupleAchievement.coupleId` |
| Notifications spam → opt-out masivo | Frequency cap 1/48h + 3 categorías separadas en Settings |
| Niveles inflados por actividad simbólica (logging mínimo repetido) | XP solo de transacciones positivas verificadas, no log creation |
| Streak rotos por bug de TZ | Test de TZ-safety en achievementEngine + UTC normalize en cron |
| Migrations Supabase fallan | Pre-flight `prisma migrate status` + rollback plan documentado |

## 13. Métricas de éxito (medible 30d post-launch)

- D7 retention: +15% sobre baseline pre-v1.7.
- Logros desbloqueados / pareja activa: ≥6 mediano (target 30d).
- Web push opt-in rate: ≥40% (industry avg 60% pero somos B2C español).
- Streak median en parejas activas: ≥7 días daily, ≥3 weekly.
- Sentiment score (frase post-mood "¿cómo te ha ido?"): ≥4.0/5.
