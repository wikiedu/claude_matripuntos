# v2.0.3 · Analytics Pro — Spec

**Fecha:** 2026-05-02
**Versión:** v2.0.3
**Branch:** `feature/v2.0.3-analytics-pro`
**Tag al merge:** `v2.0.3`

## 1. Goal

Convertir Analytics de "tres charts básicos" en una **vista narrativa** que ayude a la pareja a entender patrones y tomar decisiones — sin pretender ser una herramienta de business intelligence.

## 2. Problemas a resolver

1. Charts actuales son técnicos (saldos, transacciones); no cuentan historias.
2. No hay "insight automático" — el user debe inferir conclusiones por su cuenta.
3. No hay segmentación por categoría / dia de la semana / hora del día.
4. No hay export para llevarse los datos.
5. Comparaciones temporales ("este mes vs el pasado") inexistentes.

## 3. Pillars

- **Narrativa > números puros**: cada sección abre con una frase ("Esta semana habéis sido más activas/os por las tardes").
- **Privacy local**: cómputo cliente cuando posible para reducir round-trips.
- **Export siempre disponible** (CSV + JSON).
- **Mobile first**: charts compactos, no dashboards densos.
- **Free-tier completo en lo básico, premium para periodos extendidos**.

## 4. Scope v2.0.3

### 4.1 Insights cards (Dashboard)
- 3-5 cards rotativas con frase + cifra clave + chip de tendencia (↑/↓/=).
- Ejemplos: "Tus tardes pesan +18% que las mañanas", "La cocina representa 34% de vuestras tareas", "Domingo es vuestro día más activo".
- Generadas server-side, cache 24h.

### 4.2 Vistas analíticas (`/analytics/advanced`)
- **Time-of-day heatmap** (24h × 7d) con intensidad = #actividades.
- **Categories breakdown** stacked bar por user × categoría.
- **Mood timeline** (últimos 90d) con eventos sobre-impuestos.
- **Streak history** mini-chart.
- **Equity curve**: |saldo neto| en el tiempo, con bandas verde/rojo según pillars de equilibrio.

### 4.3 Comparaciones
- Toggle "Este mes vs anterior" en cada vista.
- Diff cards: "Hicisteis +5 tareas más", "Mood medio bajó de 'tranquilo' a 'cansado'".

### 4.4 Export
- CSV (events, taskLogs, transactions, moods, journal en archivos separados).
- JSON consolidado (igual que GDPR export pero con timestamps locales y headers humanos).
- Premium: PDF "anuario" con highlights.

### 4.5 Recomendaciones suaves (no AI)
- Heurísticas server-side: "Lleváis 3 días sin loggear nada", "El último mood fue hace 6 días", "Tu pareja registró 5 tareas que aún no verificaste".
- Rule-based, transparentes (clic → "¿Por qué veo esto?" muestra la regla).

## 5. Schema additions

```prisma
model AnalyticsInsight {
  id          String   @id @default(cuid())
  coupleId    String
  couple      Couple   @relation(fields: [coupleId], references: [id], onDelete: Cascade)
  kind        String   // 'time-of-day' | 'top-category' | 'streak-broken' | etc.
  title       String
  body        String
  payload     String   // JSON con números/refs
  trend       String?  // 'up' | 'down' | 'flat'
  generatedAt DateTime @default(now())
  validUntil  DateTime
  seenAt      DateTime?

  @@index([coupleId, validUntil])
  @@index([coupleId, kind])
}
```

## 6. API routes nuevas

```
GET  /api/analytics/insights              → cards rotativas activas
POST /api/analytics/insights/:id/seen
GET  /api/analytics/heatmap?from&to       → matriz 24×7
GET  /api/analytics/categories?period
GET  /api/analytics/mood-timeline?days=90
GET  /api/analytics/equity-curve?days
GET  /api/analytics/compare?period=month  → diffs vs período anterior
GET  /api/analytics/export?format=csv|json
GET  /api/analytics/recommendations       → reglas activas para esta pareja
```

## 7. Decisiones brainstorm

| # | Tema | Decisión | Razón |
|---|---|---|---|
| 1 | AI o reglas | Reglas server-side transparentes | Privacy + control, AI llega v3 |
| 2 | Insights frequency | Cron diario regenera + cache 24h | Coste bajo, narrativa fresca |
| 3 | Heatmap granularity | 24h × 7d | Universal, mobile-friendly |
| 4 | Comparaciones | "Este mes vs anterior" + "Esta semana vs anterior" | Suficiente para narrativa |
| 5 | Export formats | CSV + JSON free, PDF premium | Diferenciación natural |
| 6 | Mood timeline depth | 90d default, 365d premium | Free tier funcional |
| 7 | Recomendaciones | Rule-based + UI "¿por qué?" | Transparencia anti-magia |
| 8 | Equity curve bandas | ≤5 verde, ≤15 amarillo, >15 rojo | Coherente con audit thresholds |
| 9 | Privacy compute | Server-side (los datos ya están allí) | Simplifica, evita JS pesado en cliente |
| 10 | Tab nav v2.0.3 | Mantener `/analytics` con vistas + sub-rutas | Compatibilidad |

## 8. Out-of-scope

- AI summaries → v3.0
- Predicción ("la semana que viene tendréis X conflictos") → futuro lejano
- Comparación con otras parejas anónimas → privacidad arriesgada
- Real-time websocket charts → overkill

## 9. Métricas éxito 30d

- 50%+ users abren Analytics ≥1×/semana
- Insights "seen" rate ≥40%
- Export downloads ≥10% MAU
- "¿Por qué veo esto?" CTR ≥15% (señal de transparencia útil)
