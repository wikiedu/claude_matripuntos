# v1.3 · La Casa — Spec de diseño
**Fecha:** 2026-04-14  
**Tipo:** Hub doméstico · Tareas 2.0, Lista de la compra, To-dos, Digest semanal  
**Estado:** Aprobado por usuario

## Contexto

v1.3 convierte Matripuntos en un hub de gestión doméstica real. Añade planificación temporal a las tareas (recurrencia + vista semanal), una lista de la compra compartida, to-dos personales con opción de compartir, y un digest semanal in-app. La navegación no cambia (opción B aprobada): Lista de la compra y To-dos son accesibles desde el botón ➕ y desde cards en el Dashboard.

## 2. Sistema de recurrencia

### Estrategia: ventana deslizante de 8 semanas

- Al crear o editar una tarea recurrente, `generateOnCreate(task)` genera instancias (`TaskLog` con `isAutoGenerated=true`) para las próximas 8 semanas desde `recurrenceStart`.
- Cada lunes a las 08:00, `runWeeklyGeneration()` extiende la ventana una semana más para todas las parejas con tareas recurrentes activas.
- Si `maxOccurrences` está definido: se para cuando `occurrenceCount` alcanza ese valor.
- Si `recurrenceEnd` está definida: no se generan instancias más allá de esa fecha.
- Si ambos son `null`: la recurrencia es indefinida.

### Frecuencias

| Valor | Descripción |
|---|---|
| `daily` | Cada día |
| `biweekly` | Cada 2 días |
| `weekly` | Cada semana (mismo día) |
| `bimonthly` | Cada 2 semanas |
| `monthly` | Cada mes (mismo día del mes) |

### `recurringTaskService.ts`

Exporta:
- `generateOnCreate(task: Task): Promise<void>` — genera ventana inicial al crear/editar
- `generateInstancesForCouple(coupleId: string): Promise<void>` — extiende ventana una semana
- `runWeeklyGeneration(): Promise<void>` — itera todas las parejas con tareas recurrentes activas

Registrado en `server.ts` con `node-cron`:
```typescript
cron.schedule('0 8 * * 1', () => {
  recurringTaskService.runWeeklyGeneration().catch(console.error)
  digestService.sendWeeklyDigests().catch(console.error)
})
```

## 3. API

### Tareas — nuevos parámetros

```
GET  /api/tasks/logs?view=week&from=YYYY-MM-DD&to=YYYY-MM-DD
     → TaskLog[] con scheduledFor en el rango (incluye isAutoGenerated)

POST /api/tasks/:id/schedule
     body: { scheduledFor, frequency?, recurrenceEnd?, maxOccurrences? }
     → Task actualizada + genera instancias
```

El endpoint existente `POST /api/tasks/logs` acepta `scheduledFor` como campo opcional.

### Lista de la compra

```
GET    /api/shopping
       → { active: ShoppingList & { items: ShoppingItem[] },
           history: (ShoppingList & { items: ShoppingItem[] })[] }
       history = últimas 4 listas archivadas

POST   /api/shopping/items
       body: { text }
       → ShoppingItem creado en la lista activa

PUT    /api/shopping/items/:id
       body: { isChecked?, text? }
       → ShoppingItem actualizado

DELETE /api/shopping/items/:id
       → 204

POST   /api/shopping/archive
       → archiva lista activa (isActive=false, archivedAt=now), crea nueva vacía
       → nueva ShoppingList
```

Si no existe lista activa para la pareja, `GET /api/shopping` la crea automáticamente.

### To-dos

```
GET    /api/todos
       → { mine: Todo[], partnerShared: Todo[] }
       mine = todos del usuario actual (todos + shared)
       partnerShared = todos del partner con isShared=true (solo lectura)

POST   /api/todos
       body: { text, dueDate?, isShared? }
       → Todo creado

PUT    /api/todos/:id
       body: { text?, isCompleted?, dueDate?, isShared? }
       → Todo actualizado (solo el propietario puede editar)

DELETE /api/todos/:id
       → 204 (solo el propietario puede eliminar)
```

### Digest semanal

```
POST   /api/digest/weekly   (uso interno, llamado por cron)
       → crea Notification con type='weekly_digest' para ambos usuarios del couple
```

## 4. Servicio de digest (`digestService.ts`)

`sendWeeklyDigests()` — itera todas las parejas activas y para cada una:

1. Calcula la semana anterior (lun–dom)
2. Obtiene: balance de puntos de ambos usuarios, logros desbloqueados esa semana, total de tareas completadas, estado de racha diaria actual
3. Crea una `Notification` para cada usuario con:
   - `type: 'weekly_digest'`
   - `title: '📬 Tu resumen semanal'`
   - `message`: JSON con los datos del resumen (parseado en frontend para renderizar el card)

## 5. Frontend

### Componentes nuevos

#### `WeeklyTaskView.tsx`
- Rejilla de 7 columnas (lunes → domingo)
- Cada columna muestra los `TaskLog` con `scheduledFor` en ese día
- Cada item: nombre de tarea, franja horaria, puntos, estado (pendiente/verificado/disputado)
- Scroll horizontal en móvil (overflow-x: auto)
- Consume el endpoint `GET /api/tasks/logs?view=week&from=&to=`

#### `TaskScheduleForm.tsx`
- Formulario embebido en el modal de creación/edición de tarea
- Campos: fecha planificada (date+time picker), frecuencia (select: ninguna/diaria/bisemanal/semanal/quincenal/mensual), fin (radio: nunca / fecha / después de N veces)
- Solo visible si el usuario activa "Planificar esta tarea"

#### `ShoppingList.tsx` (página completa, ruta `/shopping`)
- Input rápido arriba para añadir item (Enter para confirmar)
- Lista de items: checkbox + texto + botón eliminar
- Botón "Archivar lista" (con confirmación)
- Acordeón "Listas anteriores" con las últimas 4 semanas (items en readonly)

#### `TodoList.tsx` (página completa, ruta `/todos`)
- Input rápido arriba: texto + fecha opcional + toggle "Compartir con pareja"
- Lista "Mis to-dos": items con checkbox, fecha si existe, icono 👁 si está compartido
- Sección "Compartidos por mi pareja" (read-only, solo visible si el partner tiene to-dos compartidos)

### Modificaciones en páginas existentes

#### `Tasks.tsx`
- Toggle **Lista / Semana** en el header
- En vista lista: badge `🔄 Recurrente` (purple) o `📅 Planificada` (amber) en las tareas que lo sean
- En vista semana: renderiza `<WeeklyTaskView />`
- Modal de crear/editar tarea: añade `<TaskScheduleForm />`

#### `Dashboard.tsx`
- Dos cards nuevas debajo de `<StreakWidget>`:
  - `🛒 Lista de la compra` — muestra count de items sin marcar, link a `/shopping`
  - `📝 To-dos` — muestra count de to-dos sin completar (propios), link a `/todos`

#### `BottomNav.tsx` + botón ➕
- El menú de acción rápida del ➕ añade:
  - "🛒 Añadir a la compra" → abre modal con input rápido que hace POST `/api/shopping/items`
  - "📝 Nuevo to-do" → abre modal con input rápido que hace POST `/api/todos`

### Hooks nuevos

- `useShoppingList()` — React Query, `queryKey: ['shopping']`, llama a `GET /api/shopping`
- `useTodos()` — React Query, `queryKey: ['todos']`, llama a `GET /api/todos`
- `useWeeklyTasks(from, to)` — React Query, `queryKey: ['tasks', 'week', from, to]`, llama a `GET /api/tasks/logs?view=week&from=&to=`
