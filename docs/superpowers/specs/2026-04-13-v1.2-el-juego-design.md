# v1.2 · El Juego — Spec de diseño
**Fecha:** 2026-04-13  
**Tipo:** Gamificación · Niveles, mapa de logros, rachas, reglas del juego  
**Estado:** Aprobado por usuario

---

## Contexto

v1.2 convierte Matripuntos en una app con un bucle de engagement real. El corazón es la **racha** — la tensión diaria de no romperla. Sobre esa base se construyen niveles de pareja (progreso a largo plazo), un mapa de logros visual estilo Duolingo, editor de categorías con aprobación bilateral, y un panel de Reglas del Juego.

## 1. Base de datos

### Nuevos modelos

#### `AchievementDefinition` — logros globales del sistema
```prisma
model AchievementDefinition {
  id          String   @id @default(cuid())
  name        String
  description String
  icon        String   // emoji
  rarity      String   // common | rare | epic | legendary
  category    String   // constancia | equilibrio | consenso | rendimiento | pareja | secretos
  condition   String   // JSON: { type, threshold, ... }
  xpReward    Int      @default(50)
  orderIndex  Int      @default(0)  // posición en el mapa serpentante

  coupleAchievements CoupleAchievement[]

  createdAt   DateTime @default(now())
}
```

#### `CoupleAchievement` — progreso por pareja
```prisma
model CoupleAchievement {
  id                      String   @id @default(cuid())
  coupleId                String
  achievementDefinitionId String
  
  progress                String?  // JSON: { current, target }
  unlockedAt              DateTime?

  couple                  Couple                @relation(fields: [coupleId], references: [id], onDelete: Cascade)
  definition              AchievementDefinition @relation(fields: [achievementDefinitionId], references: [id], onDelete: Cascade)

  @@unique([coupleId, achievementDefinitionId])
  @@index([coupleId])
}
```

#### `RuleProposal` — propuestas de reglas y categorías
```prisma
model RuleProposal {
  id               String   @id @default(cuid())
  coupleId         String
  proposedById     String
  
  type             String   // rule | category | category_edit
  payload          String   // JSON: campos de la regla/categoría propuesta
  proposerComment  String?
  
  status           String   @default("pending") // pending | accepted | rejected
  responderComment String?
  respondedAt      DateTime?
  respondedById    String?

  couple           Couple   @relation(...)
  proposedBy       User     @relation("ProposalsSent", ...)
  respondedBy      User?    @relation("ProposalsReceived", ...)

  createdAt        DateTime @default(now())
  updatedAt        DateTime @updatedAt
}
```

### Campos nuevos en `Couple`
```prisma

```

Los modelos `Achievement` y `UserAchievement` se mantienen por compatibilidad pero no se usan en nuevas funcionalidades.

## 2. Sistema de XP y niveles

### Fórmula XP
```
XP = (puntosHistóricos × 0.5) + (logrosDesbloqueados × xpReward) + (racha diaria × 2) + (semanasActivas × 10)
```

- `puntosHistóricos`: suma de `PointsTransaction.amount` positivas del couple
- `logrosDesbloqueados`: suma de `xpReward` de cada `CoupleAchievement.unlockedAt != null`
- `racha diaria`: `Couple.dailyStreakDays` actual
- `semanasActivas`: count de `CoupleScore` con `overallScore > 0` del couple

El XP se recalcula y persiste en `Couple.xp` tras cada: tarea completada, evento aceptado, logro desbloqueado.

### Niveles
| Nivel | Emoji | Nombre | XP mínimo |
|---|---|---|---|
| 1 | 🪺 | Nido | 0 |
| 2 | 🌿 | Brote | 300 |
| 3 | 🏡 | Hogar | 2.000 |
| 4 | 🌳 | Raíces | 6.000 |
| 5 | 💎 | Diamante | 15.000 |
| 6 | ⭐ | Leyenda | 35.000 |
| 7 | ♾️ | Eterno | 80.000 |

Al subir de nivel: notificación in-app a ambos usuarios.

## 3. Sistema de rachas

### Racha diaria 🔥
- Se incrementa si el día tiene ≥1 `TaskLog` con `status = 'verified'` o `'pending'` de cualquiera de los dos usuarios del couple.
- Se evalúa al completar una tarea (comparando `lastActivityDate` con fecha actual).
- **Congelador automático**: si falla un día y `dailyStreakFreezerUsed = false` → se activa, no se rompe la racha, se notifica a ambos. `dailyStreakFreezerUsed` se resetea a `false` cada lunes.
- Si ya se usó el congelador esa semana y falla otro día: racha se resetea a 0.

### Racha semanal ⚖️
- Se incrementa si la semana tiene `CoupleScore.equilibrium ≥ 40`.
- Se evalúa cada lunes al calcular el `CoupleScore` semanal.
- Sin congelador para la racha semanal.

### Multiplicador de puntos (racha diaria)
Aplica solo a `TaskLog.pointsFinal`:

| Días de racha | Multiplicador |
|---|---|
| 1–2 | ×1.0 |
| 3–6 | ×1.1 |
| 7–13 | ×1.3 |
| 14–29 | ×1.5 |
| 30–89 | ×1.75 |
| 90+ | ×2.0 |

### Multiplicador adicional (racha semanal)
+5% por cada semana de racha semanal, máximo +20%. Se aplica sobre el multiplicador diario.

Ejemplo: 10 días de racha diaria (×1.3) + 3 semanas de racha semanal (+15%) = ×1.495

### FactorMascotas
Aplica a puntos de tareas:
- 0 mascotas: ×1.0
- 1 mascota: ×1.1
- 2+ mascotas: ×1.2

Se lee del count de `Pet` del couple en cada cálculo.

## 4. Mapa de logros

### Diseño visual
- Mapa SVG con camino serpentante. Cada nodo es un `AchievementDefinition`.
- Nodo desbloqueado: icono coloreado (verde=común, azul=raro, púrpura=épico, amber=legendario).
- Nodo en progreso: anillo pulsante, muestra barra de progreso en card de detalle.
- Nodo bloqueado: gris, muestra "???" hasta que el anterior está desbloqueado.
- Tap en nodo: card de detalle con nombre, descripción, rareza, XP, y progreso actual.

### Categorías de logros (6)
`constancia` · `equilibrio` · `consenso` · `rendimiento` · `pareja` · `secretos`

Los de categoría `secretos` tienen nombre y descripción ocultos hasta desbloquear.

### Seed inicial (~30 logros)
Ejemplos representativos:

| Nombre | Categoría | Condición | Rareza | XP |
|---|---|---|---|---|
| Primer paso | constancia | 1 tarea completada | común | 10 |
| Racha de 3 | constancia | racha diaria ≥ 3 | común | 25 |
| Semana de fuego | constancia | racha diaria ≥ 7 | raro | 75 |
| Equilibrio perfecto | equilibrio | equilibrium ≥ 80 una semana | raro | 100 |
| Semanas de paz | equilibrio | 4 semanas consecutivas equilibradas | épico | 150 |
| Campeón del consenso | consenso | 10 negociaciones sin forzar | épico | 150 |
| Sin guerras | consenso | 0 eventos forzados en 30 días | épico | 200 |
| Centurión | rendimiento | 100 tareas completadas | raro | 100 |
| Dos meses de racha | constancia | racha diaria ≥ 60 | legendario | 500 |
| Nido de amor | pareja | alcanzar nivel Hogar | épico | 150 |
| ??? | secretos | [oculto] | legendario | 300 |

## 5. Editor de categorías con aprobación bilateral

### Flujo
1. Usuario pulsa "Proponer categoría" → formulario (nombre, emoji, tipo, puntos base, comentario)
2. Se crea `RuleProposal` con `type = 'category'`, `status = 'pending'`
3. Notificación in-app al partner: "[Nombre] propone nueva categoría: [nombre]"
4. Partner ve card con propuesta + comentario → acepta o rechaza con comentario opcional
5. Si acepta: se crea la `Category` en BD + notificación al proposer
6. Si rechaza: `status = 'rejected'` + notificación al proposer con comentario

### Edición de categorías existentes
Mismo flujo con `type = 'category_edit'`. El `payload` incluye el `categoryId` y los campos a cambiar.

### Categorías base
Las categorías base (`isCustom = false`) no se pueden eliminar, solo editar puntos base (con aprobación).

## 6. Panel Reglas del Juego

### Reglas configurables
| Regla | Default | Editable |
|---|---|---|
| Rondas de negociación gratuitas | 2 | Solo Premium |
| Tiempo para verificar tareas | 24h | Sí |
| Factor mascotas en tareas | según `Pet` count | Sí (override manual) |
| Multiplicador racha activo | true | Sí |

### Flujo de propuesta de regla
Idéntico al de categorías: `RuleProposal` con `type = 'rule'`, payload JSON con la regla y nuevo valor, comentario obligatorio, aprobación bilateral con comentario.

### Acceso
Settings → sección "Reglas del Juego"

## 7. API

### Nuevos endpoints

```
GET  /api/gamification/status
     → { xp, level, levelEmoji, nextLevel, xpToNext, xpProgress,
         dailyStreak, weeklyStreak, dailyMultiplier, weeklyBonus,
         freezerAvailable, lastActivityDate }

GET  /api/achievements/map
     → [ { id, name, description, icon, rarity, category, xpReward,
           orderIndex, status: 'unlocked'|'in_progress'|'locked',
           unlockedAt?, progress?: { current, target, percentage } } ]

POST /api/achievements/check
     → { newlyUnlocked: AchievementDefinition[] }
     (llamado internamente — no expuesto al cliente directamente)

GET  /api/rules
     → { rules: [ { key, value, description } ],
         proposals: [ RuleProposal ] }

POST /api/rules/propose
     body: { type, payload, comment }
     → RuleProposal

PUT  /api/rules/:id/respond
     body: { status: 'accepted'|'rejected', comment? }
     → RuleProposal actualizado

POST /api/categories/propose
     body: { name, emoji, type, basePoints, comment }
     → RuleProposal

PUT  /api/categories/:id/propose-change
     body: { ...fields, comment }
     → RuleProposal
```

## 8. Frontend

### Componentes nuevos
- `StreakWidget.tsx` — dos contadores (🔥 diario + ⚖️ semanal) con multiplicador actual y dots de los últimos 7 días. Ubicación: Dashboard debajo del balance.
- `LevelProgress.tsx` — barra XP + nivel actual (emoji + nombre) + XP hasta el siguiente. Ubicación: Dashboard y página Logros.
- `AchievementsMap.tsx` — mapa SVG serpentante. Página completa. Incluye filtro por categoría y card de detalle al pulsar nodo.
- `RuleProposalCard.tsx` — card de propuesta pendiente: descripción del cambio propuesto, comentario del proposer, botones aceptar/rechazar con textarea de comentario.

### Hooks nuevos
- `useGamificationStatus()` — React Query, llama a `GET /api/gamification/status`. Se invalida tras mutaciones de `taskLogs` y `events`.
- `useAchievementsMap()` — React Query, llama a `GET /api/achievements/map`.

### Páginas modificadas
- `Dashboard.tsx` — añadir `<StreakWidget>` y `<LevelProgress>` debajo del balance + frase del día.
- `Achievements.tsx` — reemplazar lista actual por `<LevelProgress>` arriba + `<AchievementsMap>`.
- `Settings.tsx` — nueva sección "Reglas del Juego" con `<RuleProposalCard>` para propuestas pendientes.
- `Tasks.tsx` — al completar tarea, invalidar query `gamification`.

### Paleta de rareza

- Raro: `#60a5fa` (azul)
