# v2.0.2 · Journaling — Spec

**Fecha:** 2026-05-02
**Versión:** v2.0.2
**Branch:** `feature/v2.0.2-journaling`
**Tag al merge:** `v2.0.2`

## 1. Goal

Añadir capa narrativa al producto: una **bitácora de pareja** donde queden registradas no solo las acciones (puntos, tareas) sino también los momentos, reflexiones y aprendizajes. Punto de inflexión: Matripuntos pasa de "app de tareas gamificada" a "memoria viva de la pareja".

## 2. Problemas a resolver

1. **El historial es solo cuantitativo**: el user ve "47 actividades aceptadas" pero no recuerda qué pasó realmente.
2. **No hay espacio para sentimientos**: mood vigente caduca a las 24h. Las reflexiones más profundas se pierden.
3. **Conflictos no quedan documentados**: una negociación dura, una disputa de tarea, un momento difícil no tienen lugar para procesarse en la app.
4. **Hitos privados se olvidan**: "el primer fin de semana solos sin hijos", "la cena en X restaurant" — nada de esto queda registrado.
5. **Retrospectiva inexistente**: no hay forma de revisar "¿cómo nos fue este mes/trimestre/año?".

## 3. Pillars de diseño

- **Privacidad granular**: cada entry es individual privada, compartida con pareja, o pública (a la app — read-only para el partner).
- **Sin presión de escribir**: prompts opcionales, voice notes, foto-only entries.
- **Retrospectiva natural**: vistas por semana/mes/año con highlights automáticos.
- **No-AI por defecto**: las prompts y resúmenes son humanos en v2.0.2. AI llega en v3+.
- **Export friendly**: bundle JSON + PDF "anuario" exportable (incluido en GDPR Art. 20 export).

## 4. Scope v2.0.2

### 4.1 Tipos de entry

- **Reflexión**: texto libre, hasta 5000 chars. Privada o compartida con pareja.
- **Foto/galería**: hasta 4 imágenes con caption. Compartida.
- **Voice note**: hasta 3 minutos audio. Privada (user) o compartida.
- **Hito**: marca un momento especial con título + fecha + nota corta. Compartida.
- **Carta**: mensaje específico de un user al otro. Lectura asíncrona, notificación al partner.

### 4.2 Prompts diarios opcionales

- 1 prompt por día generado del catálogo (~200 prompts curados): "¿Qué te ha hecho sonreír hoy?", "¿Algo que tu pareja hizo te sorprendió?", "Una cosa que aprendisteis juntos esta semana".
- Notification suave 1×/día (opt-in) → tap → prompt + form.
- Skip libre, no penaliza.
- Catálogo rotativo: no repetir mismo prompt en 30d.

### 4.3 Retrospectivas

Generadas server-side cada lunes (semana anterior), 1° de mes (mes anterior), 1 enero (año anterior):

```
Esta semana en vuestra pareja:
- 12 actividades aceptadas
- Saldo final: equilibrado (-2 puntos)
- 3 reflexiones escritas, 1 hito marcado
- Mood predominante: tranquilos
- Foto destacada: 📸 [thumbnail]

Recuerdos que escribisteis:
> "Cena del sábado increíble"
> "Anna hizo algo que…"
```

Visible en `/journal/retrospective` + push notification con link.

### 4.4 Vistas

- **Feed cronológico**: tipo Twitter inverso, scroll infinito por mes.
- **Calendar overlay**: hover en `/calendar` → muestra entries de ese día.
- **By-tag**: tags libres opcionales en cada entry para indexar.
- **Anuario**: scroll vertical visual con un "spread" por mes, highlights + foto.

### 4.5 Sharing model

- Por defecto: privadas (solo escritor las ve).
- Toggle "compartir con pareja" al crear/editar.
- Pareja puede leer, no editar. Puede reaccionar con emoji (1 emoji por entry).
- Comentar partner: opcional v2.0.3 (out-of-scope ahora).

### 4.6 Storage de archivos

- Imágenes: Supabase Storage bucket privado, signed URLs con TTL 1h.
- Audio: misma estrategia. WebM Opus, max 5MB por entry.
- CDN: cache via Render edge (post v3.x).
- Cost cap: 100MB/couple en free tier (≈400 fotos + 30 audios). Premium: 5GB.

## 5. Data model

```prisma
model JournalEntry {
  id          String   @id @default(cuid())
  coupleId    String
  couple      Couple   @relation(fields: [coupleId], references: [id], onDelete: Cascade)
  authorId    String
  author      User     @relation(fields: [authorId], references: [id])
  type        String   // reflection | photo | voice | milestone | letter
  title       String?
  body        String?  // texto markdown limitado
  shared      Boolean  @default(false)
  attachments String   // JSON [{ url, type, size, caption?, durationSec? }]
  tags        String   // JSON array of strings
  promptId    String?  // si se respondió a un prompt
  prompt      JournalPrompt? @relation(fields: [promptId], references: [id])
  recipientId String?  // para tipo 'letter', destinatario
  recipient   User?    @relation("JournalRecipient", fields: [recipientId], references: [id])
  readByPartnerAt DateTime?  // tracking lectura
  createdAt   DateTime @default(now())
  updatedAt   DateTime @updatedAt

  reactions   JournalReaction[]

  @@index([coupleId, createdAt])
  @@index([authorId, createdAt])
}

model JournalReaction {
  id        String   @id @default(cuid())
  entryId   String
  entry     JournalEntry @relation(fields: [entryId], references: [id], onDelete: Cascade)
  userId    String
  user      User     @relation(fields: [userId], references: [id])
  emoji     String
  createdAt DateTime @default(now())

  @@unique([entryId, userId, emoji])
}

model JournalPrompt {
  id        String   @id // 'q-001'
  text      String
  category  String   // reflection | gratitude | future | conflict | celebration
  weight    Int      @default(1)  // probabilidad de aparición
}

model JournalRetrospective {
  id          String   @id @default(cuid())
  coupleId    String
  couple      Couple   @relation(fields: [coupleId], references: [id], onDelete: Cascade)
  period      String   // week | month | year
  startDate   DateTime
  endDate     DateTime
  data        String   // JSON con todos los stats + entries highlights
  generatedAt DateTime @default(now())
  seenByUser1 Boolean  @default(false)
  seenByUser2 Boolean  @default(false)

  @@unique([coupleId, period, startDate])
}
```

## 6. API routes

```
GET    /api/journal/entries          ?type=reflection&from=...&to=...&shared=true&tag=viaje
POST   /api/journal/entries          → crear
GET    /api/journal/entries/:id
PUT    /api/journal/entries/:id      → solo author
DELETE /api/journal/entries/:id      → solo author

POST   /api/journal/entries/:id/react      { emoji: string }
DELETE /api/journal/entries/:id/react      → quita reacción del user actual

GET    /api/journal/prompts/today    → prompt del día (deterministic seeded)
POST   /api/journal/prompts/skip     → marca prompt como saltado hoy

GET    /api/journal/retrospectives   ?period=month
GET    /api/journal/retrospectives/:id
POST   /api/journal/retrospectives/:id/seen

POST   /api/journal/uploads          → multipart, devuelve signed URL temporal
```

## 8. Decisiones brainstorm

| # | Tema | Decisión | Razón |
|---|---|---|---|
| 1 | AI o no-AI | No-AI en v2.0.2 (prompts curados manualmente) | Privacy + control narrativo |
| 2 | Storage | Supabase Storage signed URLs | Coherente con stack |
| 3 | Sharing default | Privado | Pillar privacidad |
| 4 | Prompts language | Solo español v2.0.2 | Audiencia actual |
| 5 | Voice format | WebM Opus 5MB max | Calidad/peso balance |
| 6 | Retrospective auto | Cron lunes/1mes/1ene | Predictible |
| 7 | Comments partner | Out-of-scope, solo reactions | YAGNI |
| 8 | Premium hooks | Storage cap + retrospective avanzada | Diferenciación natural |
| 9 | Export | Incluido en /api/account/export GDPR | Single source |
| 10 | Tags | Free-form, no taxonomía cerrada | Flexibilidad |

## 9. Out-of-scope v2.0.2 (futuras)

- Comentarios partner sobre entries → v2.0.3.
- AI summaries ("este mes os habéis enfocado en…") → v3.x.
- Sharing entries fuera de la pareja (familia, amigos) → v3.x con permisos granulares.
- Templates/journaling guiado (Bullet Journal style) → v3.x.
- Export PDF anuario → v3.x (HTML→PDF cost).

## 10. Riesgos

| Riesgo | Mitigación |
|---|---|
| Storage cost runaway | Hard cap 100MB free + alerta a 80% |
| Audio transcription costs si añadimos AI | Out-of-scope v2.0.2 |
| Partner read sin notificar / awkward | Toggle "marcar leída" + notification clear |
| Retention de letters (sensibles) | Misma política GDPR — incluidas en export y delete |
| Markdown injection / XSS | DOMPurify + react-markdown sanitize strict |

## 11. Métricas éxito 30d

- 25%+ users escriben ≥1 entry / semana.
- Median entries/couple/mes: ≥10.
- Retro lectura: 60%+ users abren retrospective semanal.
- Photo entries: ≥30% del total.
- Storage usage promedio: 15MB/couple (lejos del cap).

## 12. Compliance & retention

- JournalEntry incluido en retention 5 años (datos personales narrativos).
- Soft delete (deletedAt) + hard delete tras 30d (igual que User).
- Export incluye todos los attachments inline base64 o links signed (decisión final en plan).
- Voice notes no se transcriben para no introducir surface area de ASR.
