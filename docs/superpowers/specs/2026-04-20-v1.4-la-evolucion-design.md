## 1. Objetivo

Evolucionar Matripuntos visual y estructuralmente a un diseño v2 coherente end-to-end, incorporando una pantalla de Analítica dedicada (con gating Premium visible-bloqueado) y preparando terreno UX para Premium (v3.0) sin introducir cobros aún.

**Principios:**
- Dark-first: el gradient indigo es parte de la marca, no un modo.
- Jerarquía clara en Dashboard: saludo → progreso → acción (tareas) arriba del fold.
- Prioridad conceptual: el tradeo de actividades (consume MP) y las tareas puntuales/programadas (generan MP) son el núcleo; el resto amplifica.
- Todas las pantallas del bundle v2 se rediseñan en v1.4. Lo que no cabe (Google OAuth real, push notifications, Stripe) queda explícitamente para versiones posteriores.

## 2. Alcance aprobado

### In-scope

**Pantallas** (16 archivos, todas rediseñadas full):
- Dashboard · Tasks · Calendar · Analytics (nueva) — core completo
- Login · Signup · Onboarding (6 pasos) — auth
- History (absorbido como tab en Analytics) · Achievements · Settings — gestión
- Shopping · Todos · RequestActivity · RequestInbox · NotFound — flujos y secundarias

**Navegación** — nueva topología:
- Bottom nav: `🏠 Inicio · ✅ Tareas · ➕ FAB · 📅 Calendario · 📊 Analítica`
- Header ⋯ menu: Logros · Perfil · Pareja · Reglas · Ajustes · Ayuda · Logout
- FAB sheet: 📅 Actividad → `/request-activity` · 🛒 Compra → add rápido shopping · 📝 To-do → add rápido todo

**Tema:** dark-only. Se elimina el toggle `theme` del store y las ramas light de componentes.

**Backend** — 4 endpoints analytics nuevos + 1 tabla Premium teaser (detalles §11).

**Tokens:** se adoptan los del bundle (`colors_and_type.css`) vía `tailwind.config.js` extend + `globals.css` para el gradient bg.

### Out-of-scope (para versiones posteriores)

- Google / Apple OAuth real → v2.1 Conectados. En v1.4 los botones son visuales placeholder disabled.
- Stripe / cobros reales → v3.0 Premium. En v1.4 el CTA "Hazte Premium" captura email de interés.
- LLM-generated insight → v3.0. En v1.4 el "insight mensual" es heurístico, compuesto desde plantilla.
- Push notifications → v2.1.
- Modo claro / warm light → v1.5+.
- Google Calendar sync, export PDF, referidos → v2.1 / v3.0.

## 5. Dashboard condensado

### Orden vertical

```
[Header sticky]
├─ DailyPhrase          ← 1 línea cursiva purple suave
├─ BalanceLevelHero     ← balance + nivel fusionados, gradient indigo→purple, radius 18
├─ StreakStrip          ← fila única: 🔥 N días · ×multiplicador · 🧊 congelador
│
├─ ══════ ABOVE THE FOLD termina aquí ══════
│
├─ TodayTasksSection    ← "Hoy tienes 3 tareas", TaskItems grandes con CTA amber
├─ RecentMovements      ← 3 items, compacto (link a /analytics?tab=movements)
└─ ShoppingPreview + TodoPreview  ← 2 cards horizontales, solo si hay pendientes
```

### BalanceLevelHero

Gradient `linear-gradient(135deg,#4f46e5,#7c3aed)`, radius 18, padding 16.

Top row:
- Label uppercase "BALANCE DE LA SEMANA" (11px, 0.9 opacity)
- Valor: `{balance >= 0 ? '+' : ''}{balance.toFixed(1)} MP` (34px, 800 weight, tabular-nums)
- Frase estado: "🎉 Vas 2.3 MP por delante" | "💪 Carlos va 2.3 MP por delante" | "🤝 Estáis empatados"
- Lateral derecho: mini panel `{youName}/+X · {partnerName}/+Y`

Divider (1px white 0.18) + bottom row:
- `👑 Nivel {N} · {levelName}` · `{current}/{needed} MP`
- ProgressBar (white on rgba(0,0,0,0.2)), con shimmer sutil al cargar

### StreakStrip

Una fila horizontal, padding 10, bg `rgba(168,85,247,0.08)`, border purple subtle, radius 10.

`🔥 {streak} días seguidos · ×{multiplier} · 🧊 {freezes}/1`

Si streak = 0 → "💤 Empieza una racha hoy" (sin multiplicador).

### TodayTasksSection

Card grande, título "Hoy" (15px bold) + contador "`{N}` tareas pendientes".

Cada TaskItem:
- Emoji tile 40×40 con bg de categoría
- Nombre de la tarea + assignee + MP esperados
- CTA full-width:
  - Si es asignada a mí: `[✓ Marcar como hecha]` amber gradient
  - Si es del partner: `[Esperando a {partnerName}]` outline purple disabled
  - Si es libre (primero que la haga): `[Hacerla yo]` purple gradient

Empty state: "🎉 Día libre · Has completado todo · Disfrutad el resto del día 💕"

### RecentMovements

3 items max, compacto. Card discreto, `bg: rgba(26,16,53,0.3)`.

Cada item: `{avatar} {user} · {action} · {+/-MP} · {when}` en una línea.

CTA inferior: "Ver historial completo →" → `/analytics?tab=movements`.

Si no hay movimientos en 48h → sección oculta.

### ShoppingPreview + TodoPreview

Dos cards horizontales (flex-1 cada una):
- 🛒 Compra · "`{N}` items pendientes" → `/shopping`
- 📝 To-dos · "`{N}` por hacer" → `/todos`

Ocultar si ambos están vacíos.

### Tab Básico (4 gráficos)

1. **📊 Puntos por día** — Barras semanales Tú (purple grad) vs Partner (amber grad), 7 columnas L-D. Totales bajo divider.
2. **🥧 Categorías** — Pie SVG 120px + leyenda `emoji + nombre + %` lateral.
3. **📈 Evolución del balance** — Line chart 4 semanas con área gradient, puntos + etiquetas, línea zero dashed, "↗ Tendencia: +X MP en 4 semanas" (verde si >0, amber si <0).
4. **⏱️ Tiempo invertido** — Valores Tú/Partner grandes + barra horizontal dividida, "⚖️ Reparto equilibrado — diferencia de {X}h".

### Tab Avanzado (5 gráficos)

3. **⚖️ Índice de equidad** — Gauge circular 120×120, número central grande (0-100), gradient verde→purple. Footer "↗ +4 puntos vs mes anterior".
4. **🏅 Top categorías por persona** — Barras bidireccionales purple (Tú) ↔ amber (Partner), 4-5 categorías.
5. **💬 Insight mensual** — Card gradient suave con texto heurístico + chips de destacados (`✓ +12% equidad` verde, `⚠ Mascotas desequilibrado` amber).

Posición absolute, inset 0, padding-top 80px. Gradient oscuro arriba → opaco abajo.

- CTA "👑 Hazte Premium · 3,99€/mes" amber grad full-width 44px → abre `PremiumInterestModal`

Filtros horizontales (chips scroll):
- Persona: Todos · Tú · Partner
- Categoría: Todas · 🍳 · 🚿 · 🧹 · ...

- ItemRow: `{avatar mini} {name} {action} · +{MP} · {when}`

## 7. Auth flow

### Login (`/login`)

Layout centrado vertical, padding 24, min-height 100%.

- Logo 72×72 gradient amber→purple, radius 20, 💕 emoji 36px
- Título "Matripuntos" 26px bold
- Subtítulo "Equilibrio en pareja" 13px secondary
- Input Email + Input Password (con 👁 toggle show)
- Link "olvidé mi contraseña →" (preparado, abre modal de solicitud email — backend ya soporta en v1.4 solo placeholder, deferred a v2.1)
- CTA "[Entrar]" amber full-width
- Divider "o" con líneas
- `[continuar con Google]` outline disabled + tooltip "Disponible pronto"
- `[continuar con Apple]` outline disabled + tooltip "Disponible pronto"
- Footer "¿Primera vez aquí? **Crea tu cuenta →**" → `/signup`

### Signup (`/signup`)

Wizard 2 pasos con progress bar superior + back button.

**Paso 1 — Crea tu cuenta:**
- Email · Password (min 6) · Confirm password · Checkbox "Acepto términos"
- Validación inline, botón disabled si invalid
- CTA "Siguiente →"

**Paso 2 — Cuéntanos de ti:**
- Nombre (min 2 chars)
- CTA "Crear cuenta →" → POST `/auth/register` → JWT → redirect `/onboarding`

### Onboarding (`/onboarding`) — 6 pasos

Progress bar continua 4px altura purple con fill amber.

1. **Bienvenida** — Ilustración grande (gradient tile 💕), título "Bienvenida a Matripuntos" + subtítulo "Un juego para repartir la vida en pareja". Botón "Empezar →".

2. **Tu perfil** — Input nombre + AvatarSelector (grid 6×3 emojis + color picker 8 swatches). Botón "Siguiente →".

3. **Invitar a tu pareja** — 3 radio cards:
   - 📧 **Enviar invitación por email** → input email, envía Invitation + crea Couple pendiente
   - 🔑 **Introducir código de invitación** → input 8 chars, valida con `/api/invitations/accept` y vincula a Couple existente
   - 🚶 **Empezar sola/o** → crea Couple con 1 user. Puede invitar después desde Settings.
   
   Si el user llegó con ?token=... en URL → este paso se salta automático (ya vinculado).

4. **Reglas de puntos** — 2 sliders con valores default y helper text:
   - Multiplicador nocturno (1.2–1.6, default 1.5)
   - Bonus fin de semana (0%–50%, default 25%)
   Guarda a `Configuration.multipliersConfig`.

5. **Categorías activas** — Grid 3 columnas con 9 chips checkbox, todas pre-marcadas:
   🍳 Cocina · 🚿 Baños · 🧹 Limpieza · 🛒 Compra · 📅 Logística · 👶 Cuidado · 🔧 Mantenimiento · 🌱 Jardinería · 🐕 Mascotas
   Input "+ Añadir categoría custom" al final.

6. **¡Listo!** — Resumen con avatar, nombre, estado pareja ("Vinculado con Carlos" / "Invitación enviada a X" / "Solo/a por ahora"). CTA "Entrar a Matripuntos 💕" → `/dashboard`.

Al completar → `PATCH /api/profile/me { hasCompletedOnboarding: true }`.

## 11. Backend — cambios

### Nuevos endpoints

Todos bajo `/api/analytics/` salvo el de Premium interest.

```ts
// 1. Tiempo invertido estimado
GET  /api/analytics/time-invested?range=week|month
→ { you: { hours: 4.5 }, partner: { hours: 5.2 } }

// Heurística:
// - TaskLog: duración estimada fija por categoría (config):
//   cocina 1h, limpieza 1.5h, compra 1h, cuidado 2h, baños 0.5h, ...
// - Event: (dateEnd - dateStart) en horas reales.
// - Suma por user, filtra por range.

// 2. Heatmap actividad
GET  /api/analytics/heatmap?weeks=4
→ { grid: [{ dow: 0-6, bucket: 6|9|12|15|18|21, count: N }] }

// Query Prisma: TaskLog + Event donde coupleId=X, agrupados por
// dayOfWeek(completedAt || dateStart) × hourBucket, últimas N semanas.
// Normalizar a 0-4 para opacity del heatmap en frontend.

// 3. Tasa de cumplimiento
GET  /api/analytics/completion-rate?range=month
→ { you: 92, partner: 86 }

// Ratio = COUNT(TaskLog.status=verified) / COUNT(tareas asignadas al user).
// Si no hay asignación explícita, asume reparto 50/50 por categoría histórica.

// 4. Insight mensual heurístico
GET  /api/analytics/insight?month=4&year=2026
→ {
    text: "Este mes Carlos cubrió más cocina (+4 tareas); tú llevaste más logística y compras. Reparto por tiempo: 52/48.",
    bullets: [
      { tone: 'success', text: '+12% equidad' },
      { tone: 'warn',    text: 'Mascotas desequilibrado' }
    ]
  }

// Plantilla: pide métricas (top category per user, time split, equity delta,
// categoría más desigual) y las inyecta en template fijo.
// Cache in-memory Map { "coupleId-YYYY-MM" → result }, TTL fin de mes.
// Nota: en v3.0 Premium, este endpoint se upgrade opcional a LLM real.

// 5. Extensión endpoint existente
GET  /api/analytics/points-by-category?groupByUser=true
→ { cocina: { you: 18, partner: 14 }, limpieza: { you: 12, partner: 12 }, ... }
// Si groupByUser omitido → comportamiento actual (agregado). Backward compat.
```

## 16. Riesgos y mitigaciones

| Riesgo | Probabilidad | Impacto | Mitigación |
|---|---|---|---|
| Rama larga se desincroniza con main | Media | Medio | Merge desde main a la rama cada 2-3 días; hotfixes a main se cherry-pickan |
| Heatmap sin datos (pareja nueva) | Alta | Bajo | Estado vacío: "Aún no hay suficiente actividad" |
| Insight heurístico suena robótico | Media | Bajo | Varias plantillas (8-10) con variables + seed determinista por mes |
| Tailwind extend rompe componentes v1 existentes | Baja | Alto | El extend es aditivo, no sobreescribe. QA en v1 antes de borrar |
| PremiumInterest llena la DB si hay bots | Baja | Bajo | Rate-limit 5/min por IP + validación email formato básico |
| Onboarding paso "empezar solo" crea huérfanos | Media | Medio | Cleanup cron mensual: Couples con 1 user + 0 invitations tras 30 días → recordatorio por email |
| Merge a main rompe deploy Render | Baja | Alto | Staging manual: local build + smoke test antes de push |
