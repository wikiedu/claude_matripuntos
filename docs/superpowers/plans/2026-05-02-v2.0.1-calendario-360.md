# v2.0.1 Calendario 360 — Implementation Plan

> Spec: `docs/superpowers/specs/2026-05-02-v2.0.1-calendario-360-design.md`

**Goal:** Hub temporal unificado con CalendarEntry (event/task/service/birthday/holiday/external), Google Calendar import OAuth read-only, ServiceProvider con recurrencia, vistas día/semana/mes/año, auto-derivación de birthdays/holidays.

**Architecture:** Aditiva al schema. Google OAuth con refresh tokens encriptados AES-256-GCM. Cron diario sync. Holidays JSON estático.

**Tech:** Prisma + Postgres, googleapis SDK, RRULE simplificado, React + Zustand, react-query, FullCalendar-like custom views.

**Feature flag:** `CALENDAR_360_ENABLED` (default false).

## Bloque A: schema + servicios core (1.5d)

### Task A.1: schema + migration

- Extender CalendarEntry: `externalSource`, `externalId`, `recurrence`, `color`, `metadata`, `endDate`, `allDay`.
- Nuevo modelo GoogleCalendarSync: refreshToken cifrado, scope, syncEnabled, lastSyncedAt, syncWindow, filters JSON.
- Nuevo modelo ServiceProvider: name, type, recurrence, color, active, notes.
- Migration aditiva. Index en CalendarEntry(coupleId, date) + (coupleId, type).

### Task A.2: cryptoService (AES-256-GCM)

- `encrypt(plaintext, key) → string` (iv:authTag:cipherText base64).
- `decrypt(encoded, key) → string`.
- Key desde env `GOOGLE_TOKEN_ENCRYPTION_KEY` (32 bytes hex).
- 6 unit tests: round-trip, wrong key, tampered, empty.

### Task A.3: recurrenceService (RRULE simplificado)

- Soporte: `FREQ=DAILY|WEEKLY|MONTHLY|YEARLY`, `INTERVAL`, `UNTIL`, `COUNT`, `BYDAY` (lunes-domingo subset).
- `expandRecurrence(rrule, start, windowDays) → Date[]` (cap 365 ocurrencias).
- 10 unit tests: weekly Mon/Wed/Fri, monthly cuarto martes, until cap.

### Task A.4: holidaysService + holidays-es-2026.json

- JSON con 14 holidays España 2026 (Año Nuevo, Reyes, Viernes Santo, etc.).
- `loadHolidays(year, country='es') → Holiday[]` con i18n title.
- `autoCreateHolidaysForCouple(coupleId, year)` — upsert CalendarEntry type=holiday.
- 4 tests: load shape, idempotencia upsert, año fuera de catálogo, country fallback.

### Task A.5: birthdaysService

- `deriveBirthdaysFromCouple(couple, year) → CalendarEntryDraft[]` — Child.dateOfBirth + couple anniversary.
- `syncBirthdays(coupleId)` — borrar previos + insert nuevos.
- 5 tests.
