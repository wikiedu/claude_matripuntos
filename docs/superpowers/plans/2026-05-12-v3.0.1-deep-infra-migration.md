# v3.0.1 Deep Infra Migration — Implementation Plan

> Spec: `docs/superpowers/specs/2026-05-12-mobile-pwa-strategy-design.md` (Fase 0b)
> **For agentic workers:** REQUIRED SUB-SKILL: Use `superpowers:executing-plans`. Plan operativo — alto riesgo, ejecución secuencial con verificación post-cada-tarea.

**Goal:** Migrar backend Render → Railway Hobby ($5/mes) y BD Supabase → Neon (free → Pro cuando crezca). Resolver audit S0-5 (migrations Prisma SQLite-only) regenerando baseline limpio Postgres. Configurar `api.matripuntos.com` CNAME. Eliminar deps Render+Supabase del repo.

**Architecture:** Migración escalonada con validación. Primero Railway con DATABASE_URL aún apuntando a Supabase (verificar Railway funciona). Luego cleanup migrations en Neon staging branch. Luego restore datos a Neon prod branch. Finalmente switchover DATABASE_URL → Neon, con maintenance window de ~30 min y triple dump como safety net. Rollback es instantáneo: revertir env var.

**Tech Stack:** Railway, Neon, Prisma, pg_dump/psql, Cloudflare DNS, GitHub Actions (env vars).

**Tag final:** `v3.0.1` tras merge.

**Branch:** `feature/v3.0.1-deep-infra-migration`

**Pre-requisito:** v3.0 + v3.1 + v3.2 en producción. Los usuarios pueden tener PWA instalada — la migración debe ser transparente.

### Tarea B.3 · Configurar MAINTENANCE_MODE en backend (nuevo)

Añadir un middleware que, cuando `MAINTENANCE_MODE=true`, bloquee mutations (POST/PUT/DELETE/PATCH) excepto `/api/auth/*` y `/api/health`. Sirve para la ventana de switchover.

- [ ] **Paso 1:** Crear `src/backend/src/middleware/maintenanceMode.ts`.

```ts
import type { Request, Response, NextFunction } from 'express';

const ENABLED = process.env.MAINTENANCE_MODE === 'true';
const ALLOWED_PATHS = ['/api/auth/login', '/api/auth/refresh', '/api/health'];
const SAFE_METHODS = ['GET', 'HEAD', 'OPTIONS'];

export function maintenanceMode(req: Request, res: Response, next: NextFunction): void {
  if (!ENABLED) return next();
  if (SAFE_METHODS.includes(req.method)) return next();
  if (ALLOWED_PATHS.some(p => req.path.startsWith(p))) return next();

  res.status(503).json({
    error: 'maintenance',
    message: 'Estamos mejorando la app. Vuelve en unos minutos.',
    retryAfter: 1800
  });
}
```

- [ ] **Paso 2:** Montar en `src/backend/src/server.ts` antes de rutas.

```ts
import { maintenanceMode } from './middleware/maintenanceMode';
// ...
app.use(maintenanceMode);
```

- [ ] **Paso 3:** Tests.

```ts

import request from 'supertest';
// ...
test('blocks POST when MAINTENANCE_MODE=true', async () => {
  process.env.MAINTENANCE_MODE = 'true';
  const res = await request(app).post('/api/events').send({});
  expect(res.status).toBe(503);
  expect(res.body.error).toBe('maintenance');
});
```

- [ ] **Paso 4:** Commit.

## Rollback

### Si falla en D.2.4 (restore a Neon main)

- Mantener Supabase como BD activa. NO cambiar `DATABASE_URL` en Railway todavía.
- Investigar el error de restore (probable: tipo de columna, default value, FK orden).
- Si el problema requiere cambios en schema → hacer en una rama Neon nueva, re-test.
- Mientras tanto, en Render aún funciona el backend. Quitar MAINTENANCE_MODE en Render para que app vuelva a estar disponible.

### Si falla en D.2.7-D.2.10 (Railway con Neon)

- Cambiar `DATABASE_URL` en Railway de vuelta a Supabase. Redeploy (< 60 s).
- Render sigue activo y funcional. Si urgente, apuntar DNS API de vuelta a Render (CF DNS, instant).
- App vuelve a estado pre-switchover en < 5 min.

### Si problema descubierto > 1h tras switchover (rows divergentes, datos perdidos)

- Tenemos `supabase-final.dump` con datos hasta el momento del switchover.
- Tenemos los `pg_dump` periódicos durante la ventana (D.2.3).
- Worst case: restore desde dump a una rama Neon nueva, comparar contra prod actual, mergear cambios manualmente.

### Render decomission

NO borrar el service Render hasta T+30 días. Está pagado $7/mes pero es el safety net. Tras 30 días sin incidentes, downgrade a Free y eventualmente delete.

### Supabase decommission

Igual: pausar el project a T+7d, no borrar hasta T+30d.
