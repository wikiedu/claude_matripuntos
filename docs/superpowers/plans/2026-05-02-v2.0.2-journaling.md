# v2.0.2 Journaling — Implementation Plan

> Spec: `docs/superpowers/specs/2026-05-02-v2.0.2-journaling-design.md`

**Goal:** Bitácora pareja con entries (reflexión/foto/voice/hito/carta), prompts diarios, retrospectivas auto, reactions emoji, anuario.

**Architecture:** 4 modelos schema (JournalEntry, JournalReaction, JournalPrompt, JournalRetrospective). Storage Supabase signed URLs (out-of-scope inicial: solo texto). Cron retrospective semanal/mensual/anual.

**Feature flag:** `JOURNAL_ENABLED` default true (release shape igual que v1.7/v2.0.1).

---

## Bloque A: schema + servicios core (1d)

- Schema 4 modelos + migration aditiva.
- promptsService: catálogo 30+ prompts curados ES + getTodayPrompt(coupleId) determinístico (cyrb53(coupleId+dayKey)).
- retrospectiveService: pure compute para week/month/year stats.
- 25 tests hermetic.

## Bloque B: API routes (1d)

- /api/journal/entries CRUD (auth + cap 5000 chars body).
- /api/journal/entries/:id/react (emoji reaction, unique per user+emoji).
- /api/journal/prompts/today (deterministic).
- /api/journal/retrospectives (list + read + mark seen).
- 12 contract tests.

## Bloque C: frontend hooks + componentes (1d)

- useJournalEntries, useTodayPrompt, useRetrospectives.
- JournalEntryCard (compact preview con reactions).
- JournalComposer (textarea + tags).
- JournalRetrospectiveModal (stats card).
- Nueva ruta /journal.
- 15 frontend tests.

## Bloque D: cron + telemetría + E2E + deploy (0.5d)

- Cron lunes/1mes/1ene genera retrospectivas.
- 6 telemetry events.
- 4 E2E specs.
- ROADMAP/CLAUDE.md + merge + tag v2.0.2 + deploy.

---

**Total estimado: 3.5d. Comenzar Bloque A inmediatamente.**
